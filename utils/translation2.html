Ciao xxxx,

attualmente su SEE non è stato creato un bucket dedicato.

Il file viene generato dal processo applicativo, salvato nel bucket rawfilelanding e successivamente inviato a DDH via SFTP con il naming definito in fase di censimento dell’interfaccia. In seguito, il file viene trasmesso a Microdata.

È però necessario censire formalmente anche l’interfaccia relativa al file in uscita.

Per quanto riguarda la creazione di un bucket dedicato, possiamo procedere tramite Terraform, orchestrando il deploy attraverso GitHub Actions. In questo caso sarà opportuno coinvolgere il team DevOps per l’allineamento sugli standard infrastrutturali e includere la modifica nella prossima release applicativa.

Come possibile naming del bucket potremmo valutare, ad esempio: n30-reports, in linea con le convenzioni esistenti.
